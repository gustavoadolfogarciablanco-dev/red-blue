# Constitución del Proyecto: Landing Page Red and Blue

## 1. Propósito Corporativo
- Construir una landing page para Red and Blue que:
  - Transmita profesionalismo, innovación y confianza tecnológica.
  - Atraiga clientes potenciales (empresas) de forma clara y persuasiva.
  - Refleje la excelencia técnica y el enfoque minimalista de la empresa.
  - Mantenga la confidencialidad de los clientes actuales.

## 2. Principios de Experiencia de Usuario (UX)
- Experiencia intuitiva, clara y consistente en todos los dispositivos.
- Jerarquía visual definida, facilitando la navegación y el enfoque en CTA.
- Formularios y puntos de contacto visibles y estratégicos.
- Internacionalización habilitada (multi-idioma).
- Accesibilidad mínima WCAG AA: contraste, navegación por teclado y legibilidad.

## 3. Principios de Interfaz y Diseño (UI)
- Estética corporativa, elegante y moderna, evitando clichés de diseño genérico.
- Tres modos de tema: light, dark y automático según sistema del usuario.
- Paleta de colores definida mediante CSS variables con acentos de alto contraste.
- Tipografía profesional: display font distintiva y body font refinada; no usar fuentes genéricas como Inter, Roboto o Arial.
- Layout dinámico: composición asimétrica, negative space generoso, overlap y flujo diagonal.
- Fondos con profundidad: grain overlay, gradient mesh o noise texture.
- Micro-interactions en hover states, scroll-triggered animations y staggered reveals para carga de elementos.
- Dirección estética clara y consistente: profesional, minimalista, moderna y memorable.

## 4. Principios Técnicos y Arquitectura
- Frontend moderno: HTML, CSS y JavaScript, optimizado para deploy en Vercel.
- Framework ligero permitido solo si agrega valor sin comprometer performance.
- Performance máxima: carga <2s, Lighthouse score >90.
- Optimización de assets y recursos.
- Backend mínimo y seguro para formularios y comunicación con clientes.
- Código modular, documentado y mantenible.

## 5. Contenido y Flujo
- Secciones esenciales:
  1. Hero con mensaje central: "Desarrollo seguro, rápido y escalable"
  2. Servicios y capacidades de Red and Blue
  3. Tecnologías y ventajas competitivas
  4. Formulario de contacto / WhatsApp / correo
  5. Footer corporativo
- Flujo guiado hacia CTA principal, enfocado en conversión de clientes.

## 6. Restricciones y Buenas Prácticas
- Evitar librerías innecesarias o código que afecte performance.
- Evitar estéticas genéricas o layouts predecibles.
- Todas las decisiones de diseño y UX deben alinearse con el objetivo corporativo de captar clientes.

## 7. Métricas de Éxito
- CTA visibles y funcionales.
- Experiencia fluida y rápida en todos los dispositivos.
- Feedback positivo de usuarios/clientes y métricas de engagement.
- Cumplimiento de performance y accesibilidad.
- Landing percibida como profesional, confiable, innovadora y corporativa.
